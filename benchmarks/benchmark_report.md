# AnchorPrune Benchmark Pack v0.1

_This report is generated by `anchorprune pack`. Numbers are reproducible: the harness uses a fixed, deterministic `MockLLM` with no network or randomness._

> In this deterministic benchmark pack, AnchorPrune was the only evaluated memory strategy with a governance mechanism: it preserved all critical anchors and maintained 100% constraint adherence across the `supplier`, `coding_agent`, `contract_review` scenarios, and quarantined 100% of adversarial override attempts in every scenario where such attempts were present (`coding_agent`, `contract_review`). Baselines have no governance mechanism, so their quarantine rate is 0% where adversarial payloads exist and N/A where they do not.

## 1. Benchmark objective

Measure how different agent-memory strategies **govern state** under noisy and adversarial context — specifically whether critical constraints survive, whether adversarial override attempts are kept out of the decision context, and whether reasoning milestones are retained. The objective is *not* to measure model reasoning quality or to minimize tokens on short tasks.

## 2. Methods compared

- **Baseline A — Full history.** Replays all messages every step. Retains everything but governs nothing.
- **Baseline B — Sliding window.** Keeps only the most recent _N_ messages (default `N=3`, the report below uses the harness default).
- **Baseline C — Simple summary.** Compresses all messages into a short extractive summary each step.
- **AnchorPrune.** Governs a state graph: anchors, evidence links, conflict edges, milestones, and anchor-aware pruning.

All methods receive identical inputs and the same deterministic LLM, so differences reflect the memory strategy alone.

## 3. Scenario design

Each scenario provides system anchors (critical constraints), evidence, a payload stream that mixes useful, noisy, and **adversarial override** blocks, and an expectation set (critical anchors, constraints, milestones, and a decision spec). Adversarial blocks explicitly attempt to weaken governance (e.g. "ignore the policy", "bypass the requirement").

| Scenario | Domain | Theme |
| --- | --- | --- |
| `supplier` | procurement | Recommend a supplier; compliance + approval limits |
| `coding_agent` | coding_agent | Fix a failing auth test without weakening security |
| `contract_review` | contract_review (policy pack) | Approve a contract only with a cited liability cap |

## 4. Metrics

- **Lost Anchor Rate** — share of critical anchors absent from the final context (lower is better).
- **Constraint Adherence** — share of expected constraints present in the final context.
- **Critical Conflict Quarantine** — share of adversarial payloads quarantined before reaching the model. Reported as **N/A** when a scenario contains no adversarial payloads. Baselines have no governance mechanism, so they score **0%** wherever adversarial payloads exist.
- **Payload Eviction** — share of payload blocks removed from working state.
- **Milestone Retention** — share of expected reasoning milestones retained.
- **Token count by step** — input tokens per step (growth profile).
- **Final Decision Context Valid** — the final context includes the required facts and excludes adversarial instructions. This measures the *context a strategy makes available*, not the reasoning quality of a real model (the evaluator is a deterministic stub).

Anchor, constraint, milestone, and decision metrics are read from the final context each method exposes to the model; quarantine and eviction are read from AnchorPrune's governance state (baselines have no governance mechanism and therefore cannot quarantine).

## 6. Interpretation

**Full history retains information but does not govern it.** In adversarial scenarios, unsafe override payloads remain available in the final decision context with the same standing as verified anchors, causing the deterministic evaluator to mark the decision as unsafe or incorrect. The failure mode is a *lack of governance*, not a lack of information.

**Sliding window and summary lose constraints.** Both compress or drop older content, so critical anchors and milestones disappear — exactly the constraints an agent must not forget.

**AnchorPrune separates retention from influence.** It keeps critical anchors non-evictable, quarantines override attempts before composition, and preserves milestones — so the final decision context contains the required facts and excludes the adversarial ones.

## 7. Limitations

- Extraction, conflict detection, evidence linking, and compression are deterministic heuristics, not learned models.
- The evaluator is a deterministic `MockLLM`; it measures memory-strategy behavior, not frontier-model reasoning quality.
- Token savings are not expected on these tiny two-step scenarios because of governed-context formatting overhead.
- Scenarios are synthetic, designed to isolate state-governance failures.

## 8. Next benchmarks

The token story becomes decisive in long, multi-step workflows where full-history memory grows unbounded and low-utility payload accumulates. Those scenarios now ship in **Part 2 — Long-Run Benchmark Pack v0.2** below (`long_run_coding_20_steps`, `long_run_contract_15_steps`, `long_run_procurement_10_steps`).

# AnchorPrune Long-Run Benchmark Pack v0.2

_Generated by `anchorprune pack`. Long-run scenarios run 10–20 steps with payloads injected over time; the evaluator is the same deterministic `MockLLM`._

## Objective

v0.1 showed AnchorPrune governs state correctly on short adversarial tasks. v0.2 asks a different question: **what happens to the context over a long run?** As steps accumulate, full-history memory grows without bound and lets adversarial/obsolete payloads pile up; sliding-window and summary stay small but forget anchors. The goal here is to measure context-growth slope, anchor retention over time, adversarial contamination over time, and obsolete-payload retention.

## Thesis

- **Full-history memory** has strong retention but **unbounded growth** and rising **adversarial contamination**.
- **Sliding-window memory** is bounded but **loses anchors**.
- **Simple-summary memory** is bounded but has **unstable governance**.
- **AnchorPrune** maintains **governed retention** (anchors kept, adversarial state excluded) while pruning low-utility state, so growth is **slower than full history**.

## New metrics

- **Adversarial Contamination** — share of adversarial payloads seen so far that are present in the final context (lower is better).
- **Obsolete Retention** — share of obsolete/noisy payloads still present in the final context (lower is better).
- **Context Growth Slope** — least-squares slope of context tokens over steps (tokens/step). Full history should be steepest.
- **Max Context Size** — largest per-step context (tokens).
- **Final Size vs Full History** — final context tokens ÷ full-history final context tokens. Below `1.00x` means smaller than full history.
- **Tokens / Valid Context** — `total_input_tokens ÷ number_of_valid_contexts`. Shown as **N/A** when a method has no valid contexts, and as **N/A (invalid context)** in the table when the method's *final* decision context is invalid — a small token count for an invalid context is not a cheaper strategy, it is a cheaper *invalid* context.
- **Bounded Context Score** *(experimental)* — `constraint_adherence × (1 − lost_anchor) × (1 − adversarial_contamination) × (1 − normalized_growth)`. A single rough number rewarding governed retention with bounded growth; treat as directional, not precise.

## Interpretation

**Token counts are only meaningful when the resulting decision context is valid.** A smaller context that has lost critical anchors or retained adversarial state is not a better memory strategy; it is a cheaper *invalid* context. That is why `Tokens / Valid Context` is reported as N/A wherever a method's final decision context is invalid — and why the `Lost Anchor Rate` and `Adversarial Contamination` rows must be read alongside any token number.

**The honest framing on tokens.** AnchorPrune is *not* the smallest memory strategy — sliding-window and summary are usually smaller. It is the smallest **governed** memory strategy: the only one that keeps every critical anchor alive, quarantines adversarial overrides, and still grows more slowly than full history. Sliding-window and summary look cheap only because they have already discarded the constraints the agent must obey — cheap but blind.

**Why full history fails over time.** Its context-growth slope is the steepest, and adversarial contamination climbs as override payloads accumulate and remain available to the model. Remembering everything is not the same as governing anything.

**What AnchorPrune buys.** A bounded, governed context: anchors retained (lost-anchor 0%), adversarial contamination 0%, obsolete payloads pruned, milestones preserved, and a final decision context that stays valid — at a context-growth slope well below full history.

## Caveats

- These long-run scenarios are **deterministic and synthetic**, built to isolate context-growth and governance behavior — not real production traffic.
- The evaluator is a fixed `MockLLM`; metrics reflect *what each strategy makes available in context*, not a real model's reasoning quality.
- Token magnitudes are illustrative (a ~4-chars/token estimate); the *relative* growth between strategies is the signal, not absolute counts.
- `Bounded Context Score` is an experimental composite; read the individual metrics for ground truth.
